\# InmoFlow — Portfolio de Analista de Sistemas



Sistema de gestión de alquileres para una agencia inmobiliaria de barrio en Saavedra, CABA.

Desarrollado como proyecto de portfolio para demostrar competencias de Analista Funcional / Analista de Sistemas.



\---



\## Contenido del repositorio



\### Semana 1 — Relevamiento y requerimientos

\- `01-alcance-del-sistema.md` — Documento de alcance del sistema (objetivos, actores, funcionalidades, restricciones)

\- `02-historias-de-usuario.md` — 10 Historias de Usuario con criterios de aceptación, organizadas en 5 épicas (MoSCoW)



\### Semana 2 — Modelado de procesos

\- `01-bpmn-as-is.png` — Diagrama BPMN del proceso actual (AS-IS)

\- `02-bpmn-to-be.png` — Diagrama BPMN del proceso optimizado (TO-BE)



\### Semana 3 — Base de datos

\- `schema.sql` — Script de creación de tablas en PostgreSQL (7 entidades, claves foráneas, constraints)

\- `queries.sql` — Consultas SQL con JOIN para los principales casos de uso del sistema



\---



\## Tecnologías utilizadas

\- PostgreSQL

\- bpmn.io

\- Git / GitHub



\---



\## Sobre el proyecto



InmoFlow cubre el ciclo completo de gestión de alquileres: búsqueda de propiedades, agendamiento de visitas, scoring de inquilinos, formalización de contratos y seguimiento de pagos.



El proyecto está diseñado para demostrar competencias de Level 2 y Level 3 en análisis de sistemas: relevamiento, modelado de procesos, diseño de base de datos y documentación técnica.
